# SocodingOJ

[使用链接](https://oj.socoding.cn/)。本人大学期间运维并负责部分功能二次开发的，供华南师范大学软件学院进行比赛、考试和课程学习的在线评测系统。原名 SCNUOJ。[github 开源](https://github.com/scnu-socoding/scnuoj)。

它派生自 [JNOJ v1](https://github.com/shi-yang/jnoj)，使用 PHP Yii2 和 Bootstrap，使用 MySQL，支持 Docker 部署。

功能如下：(更多功能参见我写的文档(密码5800) [出题教程](https://cloud.socoding.cn/s/yznU9)、[比赛创建教程](https://cloud.socoding.cn/s/DerSn))

1. 题目
   1. 支持 C++20, C, Java, Python
   2. 针对一题多解情况，支持 SPJ (special judge)
   3. Markdown 渲染，本地存储图片/动图
   4. 按标签分类，支持自定义题单和公开方式
   5. 非赛时可提供题解与讨论区，可看测试点部分信息
   6. 可按多种方式筛选查看提交结果
2. 比赛(/考试)
   1. 多种判题赛制(ACM/ICPC, OI, IOI)，支持分组分点给分
   2. 支持赛时答疑，公告，重判，代码打印
   3. 支持防作弊(比赛模式)，白名单，邀请码，打星参赛，参赛者管理
   4. 支持冻结榜单，滚榜
   5. 支持题册导出，批量题目管理
   6. 支持站外比赛跳转
3. 出题(/后台)
   1. 三种用户组：管理员(所有CRUD)、助教(仅查看与出题)、普通用户(仅前台Polygon出题，管理员可同步题目到后台)
   2. 支持上传测试点，或根据题解与输入生成测试点输出，导出测试点
   3. 隔离前台Polygon与后台两套题集，支持从其他题库导入
   4. 监控与全局管理(用户、题目、比赛等)、全局设置
4. 小组(/班级)
   1. 提供只给组内用户使用的比赛集
   2. 支持管理员、组员的权限隔离
5. 用户
   1. 支持通过邮箱找回密码
   2. 支持Markdown个人主页
   3. 统计(未/)通过的题目等信息
6. 其他
   1. 排行榜：按过题数为用户排名
   2. 首页公告：提供比赛等消息通知
   3. 帮助文档

在我使用与运维过程中，发现主要的不足点如下：

1. 性能问题
   1. 单个评测机，提交较多且测试点大时排队久
   2. 对提交代码超时/爆内存处理不足，曾导致服务器多次宕机
   3. 用户提交的代码存数据库且不分表 (70w份代码)
   4. 滚榜很慢
2. 功能问题 (多为老师/助教实际所需的)
   1. 缺乏对比赛结果(如选手代码)的多种导出功能
   2. 缺乏与其他 OJ (如 DomJudge) 相互导入导出或迁移题库的功能
   3. 没有内置代码查重，以及对比赛等情况的统计功能
   4. 缺乏更细的后台权限管理分组
   5. 小组比赛没有邀请码
   6. 讨论区缺乏检索，查询困难
3. UI/UX问题
   1. 题目搜索匹配差，没有分页跳到指定页和多页跳转；小组成员分页展示不正确、公共题库搜索无法翻页
   2. 滚榜界面不美观且元素错位
4. 其他问题
   1. 安全：密码容易撞库，后台admin没有CSRF，上传文件等部分认证薄弱
   2. 压缩命令依赖系统命令，跨平台弱，环境过时

# DomJudge

[官网](https://www.domjudge.org/)。正式 ACM 比赛官方 OJ。我曾经部署和使用于办校赛。

技术栈：PHP, MySQL, Bootstrap, 评测机 C/C++。Docker 部署。

主要功能是一场完整的比赛，优点是流程标准化与稳定性、安全，包括：

1. 分布式判题机和严格的安全沙箱
2. 支持交互题、SPJ，自定义判题流程
3. 允许 cli 提交代码，更成熟的打印请求、气球系统
4. 判题结果可以人工干预，支持导入题目、队伍等
5. 丰富的权限控制角色，配置选项

主要问题：

- 只支持比赛，功能单一
- 高可用为高可靠让步，选手交题后较长时间才看到结果

> 本项目主要参考 SocodingOJ 和 DomJudge，所以以下其他竞品不会分析太详细或者忽略部分竞品。

# OnlineJudge

[官方](https://github.com/QingdaoU/OnlineJudge)，最热门的开源 OJ，被多个比赛使用(如赛氪)，华南师范大学计算机学院的OJ也是他改编的(csoj.scnu.edu.cn)。其优点是生态丰富。

相比上述(如 socoding oj)，功能差异如下：

1. 题目和比赛的赛时统计更丰富，题目可标注难度
2. 更丰富的后台权限管理，支持IP段比赛限制
3. 异步判题队列，支持多种题型

技术：Django, PostgreSQL, Redis, Docker, Vue。

> 其他开源OJ：Hydro、SYZOJ (LibreOJ 开源)

# 洛谷

[官方](https://www.luogu.com.cn/)。信竞 OI 专用，商业化。

1. 丰富的题目，远程评测(其他OJ题目)
2. 官方和用户题集，以及专栏功能
3. 完善的题解审核，丰富的题解区生态，提供测试点下载
4. 提供个人和官方比赛功能
5. 提供云剪贴板，在线IDE，用户私信功能
6. 提供签到，过题统计，等级系统
7. 商业化：网校、卖书

# CodeForces

[官网](https://codeforces.com/)。竞赛用，主要用于比赛。

1. 完整的 ratings，频繁的比赛(div1-4等)
2. 题目难度分+标签、VP 功能、丰富的真题
3. Polygon出题系统、社区讨论
4. 赛时 protest 机制、hack、赛后数据点查看、教练号

> 同类：Atcoder：更加新手友好，但没有讨论区

# 杭电

[官网](https://acm.hdu.edu.cn/)。acm 竞赛用，可进行比赛。

1.  支持题目、用户、比赛的基本功能

缺点：C++ 版本老旧、输入输出严苛、UI 老旧。

> 类似特点：POJ
>
> 其他 OJ：UOJ, AcWing

# Vjudge

[官网](http://vjudge.com/)。整合各大 OJ 的中间平台。

1. 题源丰富
2. 支持自定义比赛题单

缺点：卡顿(即便有镜像站)

# 牛客

[官网](https://www.nowcoder.com/)，兼顾实习刷题和竞赛刷题。

1. 有普通站(讨论，招聘，学习，在线面试，简历)和竞赛站(OJ)
2. 提供比赛，公共题库，分类题单
3. 商业化：牛客暑期集训、会员、卖课(+书)

> 类似特点：PTA(pintia)，天梯赛使用；还有蓝桥的 OJ

# 力扣

有[中文站](https://leetcode.cn/)和[英文站](https://leetcode.com/)。适用于实习刷题。

1. 题目：标签、难度、提示、讨论、题解(含视频)
2. 做题：用例管理、时空击败排名、智能补全、复杂度分析
3. 用户：收藏、勋章、笔记、等级、统计、每日一题
4. 比赛：周赛、VP、排名
5. 社区：讨论贴、资源推荐
6. 商业化：VIP题目、功能、卖课(+书)、商城

## MarsCode

字节的OJ，现在疑似废弃了。融合了AI功能，支持AI辅助写题(陪练)。

- 按能力进度辅助练习，给解题实现思路，引导式教学
- 报错定位解释原因，分析判题结果
- 代码补全、注释生成、单测生成、问答
